# Бот «Заяви про себе»

Воронка: гайд → через 30 хв фото + оффер Модуля 1 за 129 грн → /about з повною програмою за 699 грн → після оплати особистий одноразовий лінк у закритий канал.

## Файли, які треба покласти в репозиторій

```
assets/guide.pdf   — PDF «5 шаблонів самопрезентації» (перше повідомлення)
assets/offer.jpg   — картинка для фоллоу-апу (перед текстом про 129 грн)
```
Якщо файлу немає — бот пропустить його і надішле лише текст.

## Оплата — три варіанти (бот сам обирає за заповненими змінними)

**Варіант 1 — WayForPay (рекомендовано, якщо мерчант вже є):**
1. У кабінеті WayForPay (Налаштування магазину) візьміть:
   - **Merchant login** → змінна `WFP_MERCHANT`
   - **Secret key** → змінна `WFP_SECRET`
   - домен сайту, на який зареєстрований магазин → `WFP_DOMAIN` (наприклад `zayavyprosebe.com`)
2. `PUBLIC_URL` — публічна адреса цього сервісу на Railway (див. деплой, крок 6).
3. Як це працює: жінка тисне «Оплатити 129 грн» → бот через API WayForPay створює рахунок і дає кнопку з лінком на сторінку оплати → вона оплачує карткою/Apple Pay → WayForPay надсилає підтвердження на вебхук бота → бот **автоматично** надсилає їй одноразовий лінк на канал, а вам — сповіщення про оплату. Підпис HMAC-MD5 перевіряється, дублікати колбеків ігноруються.

**Варіант 2 — Telegram Payments:** у BotFather підключіть провайдера (LiqPay/Portmone), токен у `PROVIDER_TOKEN`. Оплата всередині Telegram, теж повністю автоматично.

**Варіант 3 — статичні сторінки оплати:** лінки у `PAY_URL_129` / `PAY_URL_699`, доступ видаєте вручну через `/grant`.

## Закритий канал

1. Створіть приватний канал.
2. Додайте бота **адміністратором** з правом «Запрошувати користувачів за посиланням».
3. Id каналу: перешліть пост з каналу боту @userinfobot → id виду `-100…` → змінна `CHANNEL_ID`.
4. Бот генеруватиме кожній покупчині **особистий одноразовий лінк** (member_limit=1). Без `CHANNEL_ID` надсилається статичний `CHANNEL_LINK`.

## Команди адміна (тільки з вашого акаунта — ADMIN_CHAT_ID)

| Команда | Що робить |
|---|---|
| `/grant id або @username` | позначає оплату + надсилає особистий інвайт у канал |
| `/link @username` | надсилає свіжий інвайт-лінк |
| `/revoke @username` | видаляє з каналу та знімає позначку «оплачено» |
| `/users` | останні 20 користувачок: 💰 оплатила / 👤 ні |
| `/broadcast Текст` | розсилка тексту всім |
| reply + `/broadcast` | надішліть боту повідомлення (можна з фото), відповідьте на нього /broadcast — скопіює всім |

## Деплой: GitHub + Railway (покроково)

1. **GitHub.** Створіть новий репозиторій (можна приватний). Завантажте туди всі файли проєкту: `bot.py`, `requirements.txt`, `Procfile`, папку `assets/` з `guide.pdf` та `offer.jpg`. Найпростіше без git-команд: на сторінці репозиторію **Add file → Upload files**, перетягніть файли (папку assets створіть, ввівши в імені файлу `assets/guide.pdf`).
2. **Railway.** railway.app → **New Project → Deploy from GitHub repo** → авторизуйте GitHub і оберіть репозиторій. Railway сам визначить Python і встановить залежності.
3. **Змінні.** У сервісі відкрийте вкладку **Variables** і додайте як мінімум: `BOT_TOKEN`, `ADMIN_CHAT_ID`, `CHANNEL_ID`, а для WayForPay — `WFP_MERCHANT`, `WFP_SECRET`, `WFP_DOMAIN` (повний список у `.env.example`).
4. **Volume (обов'язково).** Правий клік на сервісі → **Attach Volume**, mount path: `/data`. Тут житиме база даних — без цього після кожного редеплою бот «забуде», хто оплатив.
5. **Домен.** Вкладка **Settings → Networking → Generate Domain**. Скопіюйте отриману адресу (типу `https://zayavy-bot-production.up.railway.app`).
6. Додайте цю адресу у Variables як `PUBLIC_URL` (потрібно тільки для WayForPay — туди приходитимуть підтвердження оплат). Сервіс перезапуститься автоматично.
7. **Перевірка.** У вкладці Deploy Logs має бути `Bot starting (polling)… payments: WayForPay`. Напишіть боту /start — має прийти гайд.
8. **Оновлення.** Будь-який пуш/завантаження файлів у GitHub → Railway автоматично передеплоїть.

## Логіка бота

- `/start` — вітання + PDF. Повторний `/start` не дублює і не перезапускає таймер.
- Через `FOLLOWUP_MINUTES` (30 хв) — картинка + оффер з кнопками: Оплатити 129, Про курс, Купити програму 699, Маю запитання (лінк на @ym2812). Оплаченим не надсилається.
- Після рестарту бот перечитує базу і перепланує невідправлені фоллоу-апи.
- PDF/фото вантажаться один раз, далі — кешований file_id.

## Що підкрутити

- Тексти — константи `TEXT_*` угорі `bot.py`; таймінг — `FOLLOWUP_MINUTES`; ціни — словник `PRODUCTS` (у гривнях).
